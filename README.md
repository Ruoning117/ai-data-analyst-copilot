# AI Data Analyst Copilot

An AI-powered Streamlit application that turns a raw CSV into a structured analysis report — combining statistical data profiling, LLM-generated insights, multivariate anomaly detection, and a natural-language Q&A interface.

---

## Features

### Overview (`pages/01_overview.py`)
Profiles the uploaded dataset and surfaces data quality issues. Per-column statistical profiling (`core/profiler.py`) computes descriptive statistics for every column — mean, median, std, skewness, and kurtosis for numeric columns; top-value frequencies and mode for categoricals; date range and most common year/month for datetimes; character-length stats for text columns; and uniqueness ratio for ID columns. Separate data quality checks report missing values (with clustered vs. scattered patterns), exact and near-duplicate rows, type violations, cardinality anomalies, whitespace / casing inconsistencies, and unexpected negative values. A semantic role system (`core/semantic_roles.py`) suppresses or downgrades issues that are technically true but not actionable for a given column type — for example, high cardinality in a free-text column is expected, not a problem.

### Insights (`pages/02_insights.py`)
Runs six statistical detections (distribution skew, high correlation, outlier prevalence, categorical dominance, date-gap irregularity, cross-column contradictions) and enriches each finding with a two-sentence business explanation and a recommended action generated by Claude. Renders a Plotly correlation heatmap and per-column distribution histograms for flagged columns.

### Ask (`pages/03_qa.py`)
Natural-language Q&A interface over the uploaded dataset. Claude generates executable pandas code from the user's question, runs it in a sandboxed namespace, makes one self-correction attempt on failure, and interprets the result in plain English. Chat history is preserved per dataset. The sidebar shows five LLM-generated suggested questions tailored to the actual schema.

### Anomalies (`pages/04_anomalies.py`)
Detects row-level anomalies using three complementary methods — IQR (univariate, distribution-agnostic), Z-score (univariate, normality-gated), and Isolation Forest (multivariate). A composite score rewards rows flagged by multiple methods. The page includes a sortable anomaly table with severity badges and score bars, a Plotly scatter with anomalous points highlighted, a per-column anomaly summary, and sidebar sliders for the Isolation Forest contamination rate and Z-score threshold that automatically trigger re-detection.

### Summary (`pages/05_summary.py`)
Aggregates outputs from all three analysis modules — data quality, insights, and anomaly detection — into a single LLM prompt. Claude returns a three-paragraph executive summary covering what the dataset contains, key findings and anomalies, and recommended next steps. The page also shows a compact metric table (health score, insight count, anomaly count, top finding, top anomaly) and a copy-ready plain-text block.

## Architecture

```
ai-data-analyst-copilot/
│
├── app.py                     # Entry point — st.navigation() routing and sidebar
│
├── core/                      # Pure analysis logic — no Streamlit code
│   ├── data_loader.py         # CSV loading, column type inference, DatasetMeta
│   ├── data_quality.py        # DataQualityReport: missing, duplicates, violations, health score
│   ├── profiler.py            # Per-column descriptive stats + dataset summary → DataProfile
│   ├── semantic_roles.py      # Column role inference and issue filter (suppress / downgrade)
│   ├── insight_engine.py      # Statistical detections + LLM enrichment → list[Insight]
│   ├── qa_engine.py           # LLM code generation, sandboxed exec, interpretation → QAResult
│   └── anomaly_detector.py    # IQR / Z-score / Isolation Forest → AnomalyDetectionResult
│
├── pages/                     # One file per Streamlit page
│   ├── 01_overview.py
│   ├── 02_insights.py
│   ├── 03_qa.py
│   ├── 04_anomalies.py
│   └── 05_summary.py
│
└── shared/                    # Cross-cutting utilities
    ├── llm_client.py          # Anthropic SDK wrapper with retry logic
    └── prompts.py             # All prompt templates in one place
```

All LLM calls go through `shared/llm_client.py`, which handles API key loading, exponential-backoff retries on rate-limit and server errors, and both plain-text and structured (Pydantic) response modes.

All prompt templates live in `shared/prompts.py` so they can be reviewed, iterated, and tested independently of business logic.

## Setup

**1. Clone the repository**

```bash
git clone https://github.com/Ruoning117/ai-data-analyst-copilot.git
cd ai-data-analyst-copilot
```

**2. Install dependencies**

With `uv` (recommended):

```bash
uv sync
```

Or with pip:

```bash
pip install -r requirements.txt
```

**3. Add your Anthropic API key**

```bash
cp .env.example .env
# Edit .env and set ANTHROPIC_API_KEY=sk-ant-...
```

**4. Run the app**

```bash
uv run streamlit run app.py
# or: streamlit run app.py
```

Open `http://localhost:8501` in your browser and upload a CSV file to get started.

## Design Decisions

### Statistical detection runs before the LLM

Every insight and anomaly is grounded in a specific statistic — a Pearson r value, an IQR multiple, an Isolation Forest score — before Claude ever sees it. This means the LLM's job is interpretation and communication, not detection. The result is reproducible: the same dataset always produces the same findings, and the LLM enriches rather than invents them. It also keeps token costs low: the prompt carries structured facts, not raw data.

### Traceability throughout

Every finding displayed in the UI carries its source statistic (e.g. `r=0.97`, `skewness=4.2`, `composite score=0.83`). Every code answer in the Q&A tab is expandable to show the exact pandas code that was executed. This traceability matters for analyst trust: a stakeholder can always audit where a number came from instead of taking the AI's word for it.

### Isolation Forest over IQR-only anomaly detection

IQR detects values that are extreme in a single column, but many real-world anomalies are only visible in combination — a transaction amount that is normal in isolation but anomalous given the account type and time of day. Isolation Forest detects these multivariate patterns by measuring how easy it is to isolate a point from the rest of the data using random splits. The app uses all three methods together (IQR, Z-score, Isolation Forest) and gives higher confidence scores to rows flagged by multiple methods, surfacing the most actionable anomalies first.

## Skills Demonstrated

| Area | Details |
|---|---|
| **Prompt engineering** | System/user prompt separation; structured JSON output via Pydantic; multi-turn correction loop; context-aware suggestion generation |
| **Pandas** | Dataset profiling, missing-value pattern detection, correlation matrices, IQR fencing, Z-score computation, sandboxed `exec` with restricted namespace |
| **scikit-learn** | `IsolationForest` with normalised score inversion and primary-driver attribution |
| **Statistical profiling** | Per-column descriptive statistics (mean, median, std, skewness, kurtosis, percentiles, value frequencies, date ranges, character lengths) computed once and shared across all modules via `DataProfile` |
| **Statistical analysis** | Skewness, Pearson correlation, normaltest gate for Z-score applicability, coefficient of variation for date-gap irregularity |
| **Streamlit multipage apps** | `st.navigation()` with per-page session state, sidebar shared across pages, cache invalidation by filename + row count |
| **Software architecture** | Clean separation of analysis logic (`core/`), UI (`pages/`), and shared utilities (`shared/`); no Streamlit imports in core modules; all prompts centralised in one file |
